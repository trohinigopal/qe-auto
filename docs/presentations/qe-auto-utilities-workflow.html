1 · End-to-end flow (recommended)

From user story → manual testcase → BDD artifacts → DOM or API materialization → pytest / runner.

2 · user_story_to_manual

Converts user story markdown → manual testcase markdown (default under ManualTestCases/en/).

python -m common.utils.user_story_to_manual -i UserStories/en/your_story.md -o ManualTestCases/en/your_story_manual.md
ArgumentShortRequiredDescription
--input-iYesPath to user story .md
--output-oNoOutput path (default: ManualTestCases/en/<stem>_manual.md)
--appNoOverride “Application under test” one-liner
--framework-rootNoFramework root if not default

Default: UserStories/en/foo.md → ManualTestCases/en/foo_manual.md when -o is omitted.

3 · manual_tc_to_bdd

Produces Features/nm/<stem>.feature, pageobject/nm/<stem>_page.py, stepdefination/nm/test_<stem>_steps.py.

python -m common.utils.manual_tc_to_bdd -i ManualTestCases/en/example_manual.md -c Netbanking -p nb_my_feature_stem --tag web
ArgumentShortRequiredDescription
--input-iYesManual testcase markdown path
--channel-cNoNetbanking (default), MobileBanking, API, CrossChannel
--prefix-pYesOutput file stem (e.g. nb_homefinance_emicalc_manual)
--tagNoGherkin tag(s); repeat for multiple. Default: web
--framework-rootNoPath to framework/

Examples

# Netbanking web
python -m common.utils.manual_tc_to_bdd -i ManualTestCases/en/nm_calculator.md -c Netbanking -p nb_homefinance_emicalc_manual --tag web

# API channel → then api_bdd_materialize
python -m common.utils.manual_tc_to_bdd -i ManualTestCases/en/api_leads.md -c API -p api_my_feature --tag api
Ensure stepdefination/conftest.py lists *_step_defs in pytest_plugins when steps are split. For UI, follow with bdd_dom_materialize.

4 · bdd_dom_materialize

Selenium DOM scan → *_dom_materialized.json → patches test_*_steps.py to call bdd_step_runner.execute_materialized_step.

python -m common.utils.bdd_dom_materialize -c Netbanking -p nb_homefinance_emicalc_manual --base-url https://example.com/ --secondary-url https://example.com/sub-page
ArgumentShortRequiredDescription
--channel-cYesNetbanking, MobileBanking, API, CrossChannel
--stem-pYesSame stem as manual_tc_to_bdd
--base-urlYesPrimary URL for DOM scan
--secondary-urlNoExtra page URL
--browserNoDefault: chrome
--headedFlagHeaded browser
--min-scoreNoDOM match threshold 0–1 (default 0.22)
--patch-onlyFlagNo browser; refresh JSON/patches only
--framework-rootNoFramework root

Iteration: full scan first → adjust --min-score or step wording → use --patch-only after editing JSON.

5 · api_bdd_materialize

Writes *_api_materialized.json and patches steps for api_bdd_step_runner.execute_api_materialized_step.

python -m common.utils.api_bdd_materialize -c API -p api_my_feature --api-base-url https://api.example.com

python -m common.utils.api_bdd_materialize -c API -p api_my_feature --api-base-url https://api.example.com --config path/to/overrides.json
ArgumentShortRequiredDescription
--channel-cNoDefault API; same choices as other tools
--stem-pYese.g. api_my_feature
--api-base-urlYesAPI base URL
--configNoOptional JSON merged into contract
--framework-rootNoFramework root

Secrets at runtime: API_TOKEN, API_KEY — see TECHNICAL.md §5.3.

6 · Run tests & optional runtime self-heal

Unified runner

python runner.py --platform web --tests-path Netbanking/stepdefination/nm/test_nb_homefinance_emicalc_manual_steps.py
python runner.py --platform api
python runner.py --platform cross --tests-path CrossChannel/stepdefination/nm/test_CC_authentication_cross_login_steps.py

Pytest direct (real URLs)

python -m pytest Netbanking/stepdefination/nm/test_nb_homefinance_emicalc_manual_steps.py --platform web --base-url https://your-app/
python -m pytest API/stepdefination/nm/test_api_my_feature_steps.py --platform api --api-base-url https://your-api/
VariablePurpose
SELF_HEAL=1Runtime DOM self-heal
HEADLESS=0Headed debugging
ALLURE_RECORD_VIDEO=1Playwright video under session media/videos/

bdd_dom_materialize is the main offline locator pipeline; SELF_HEAL complements for flaky UIs.

8 · Quick checklist

  • user_story_to_manual → manual TC under ManualTestCases/en/
  • manual_tc_to_bdd → feature, page object, test_*_steps.py; stem consistent
  • Web: bdd_dom_materialize with --base-url / --secondary-url
  • API: api_bdd_materialize with --api-base-url (+ --config if needed)
  • stepdefination/conftest.py → pytest_plugins for *_step_defs
  • pytest or runner.py with correct --platform and URLs/secrets
